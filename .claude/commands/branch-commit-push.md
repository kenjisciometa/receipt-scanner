# 新規ブランチ作成 & コミット & プッシュ

現在のブランチを起点に新しいブランチを作成し、変更をコミットしてリモートにプッシュしてください。

## 手順

1. `git status` で現在の変更を確認する
2. `git diff` でステージ済み・未ステージの変更内容を確認する
3. `git branch --show-current` で現在のブランチ名（ベースブランチ）を確認する
4. `git log --oneline -5` で直近のコミットメッセージのスタイルを確認する
5. 変更内容から適切な新ブランチ名を考案する（例: `feat/xxx`, `fix/xxx`, `chore/xxx`）。ユーザーから指定があればそれを使用する
6. `git checkout -b <new-branch>` で現在のブランチを起点に新しいブランチを作成する
7. **サブモジュール（他リポジトリに紐づくディレクトリ）は絶対にステージしない**。`git diff --name-only` の結果からサブモジュールのパスを除外すること
8. サブモジュール以外の全変更ファイルを個別に `git add` でステージする
9. 変更内容を分析し、リポジトリのコミットスタイルに合った簡潔なコミットメッセージを作成する
10. コミットを作成する（Co-Authored-Byトレーラー付き）
11. `git push -u origin <new-branch>` で新ブランチをリモートにプッシュし、upstreamを設定する
12. 作成したブランチ名・コミットハッシュ・プッシュ結果を報告する

## 注意

- CLAUDE.mdのGitルールに従う（サブモジュールには触れない）
- `/commit-push` とは異なり、このコマンドは**必ず新しいブランチを作成**する
- ブランチ名は命名規則（`feat/`, `fix/`, `chore/`, `docs/` など）に従う
