# コードレビュー

現在の作業ツリーの変更差分、またはmainブランチからの差分をレビューしてください。

## 手順

1. `git diff` で未ステージの変更を確認する
2. `git diff --cached` でステージ済みの変更を確認する
3. mainブランチと差分がある場合は `git diff main...HEAD` でブランチ全体の変更も確認する
4. 変更されたファイルの内容を読み、以下の観点でレビューする:
   - **バグ**: ロジックエラー、off-by-one、null/undefined参照
   - **セキュリティ**: インジェクション、認証・認可の漏れ、機密情報のハードコード
   - **パフォーマンス**: N+1クエリ、不要なループ、メモリリーク
   - **型安全性**: TypeScriptの型エラー、any型の乱用
   - **設計**: 責務の分離、重複コード、命名の一貫性
5. 問題があればファイルパスと行番号付きで具体的に指摘する
6. 問題がなければ「LGTM」と報告する
