## 2. Phase 1: ホームマーケット（フィンランド）

### 市場環境
| 項目 | 詳細 |
|------|------|
| レストラン数 | 〜12,000-15,000（推定） |
| 財務コンプライアンス | **なし**（欧州最低の障壁。TSEやレジ認証不要） |
| 主要デリバリー | **Wolt**（70-80%シェア、ほぼ独占。Foodora 2026年2月撤退） |
| 主要決済 | Vipps MobilePay、カード決済、現金極少 |
| 主要競合 | Restolution（4,000+店舗）、Solteq、Zettle |
| 会計連携 | Procountor、Netvisor（**対応済み ✅**） |

### SciPOSの競争ポジション
```
        機能充実度 高
            │
 SciPOS ★  │
            │  Restolution
            │
 ───────────┼───────────
            │
    Zettle   │  Solteq
            │
        機能充実度 低
 ←─ 低価格 ─┼─ 高価格 ─→
```

### 優先アクション
1. **Wolt連携**（フィンランド市場で最も重要なデリバリーパートナー）
2. **Restolutionからのスイッチング訴求**（無料POS + 統合型の優位性）
3. **ビュッフェ/食べ放題レストランへの集中営業**（ユニーク機能で差別化）

## 3. Phase 2: 北欧拡大（デンマーク → スウェーデン → ノルウェー）

### 3.1 デンマーク（最初の海外市場として推奨）

| 項目 | 詳細 |
|------|------|
| 財務コンプライアンス | **ソフトウェアベース**（ハードウェア不要、OCES電子署名+SAF-T） |
| 主要デリバリー | Wolt、Just Eat |
| 主要決済 | MobilePay（人口の90%が利用）、Dankort |
| 主要競合 | Lightspeed、Trivec、Zettle |
| 必要な対応 | デンマーク語ローカライズ、OCES署名、SAF-T出力 |

**なぜデンマークが最適か:**
- フィンランドと同様のソフトウェアベース財務要件（ハードウェア認証不要）
- MobilePay = フィンランドと同じVipps MobilePay基盤
- Woltが主要デリバリー = フィンランドと同じ連携で対応可能
- 文化的・ビジネス的に北欧圏で最もアクセスしやすい

### 3.2 スウェーデン

| 項目 | 詳細 |
|------|------|
| 財務コンプライアンス | **Skatteverket認証レジ + 認定制御装置が必須**（SKVFS 2014:9） |
| 主要デリバリー | Wolt、Foodora |
| 主要決済 | **Swish**（銀行振込の39%がSwish）、カード |
| 主要競合 | **Trivec/Caspeco**（8,000+顧客、2025年統合）、Extenda Retail、Zettle |
| 必要な対応 | Skatteverket認証取得、Swish決済連携、スウェーデン語対応 |

**課題:** Trivec/Caspecoが2025年統合でPOS+予約+スタッフ+分析のオールインワンを形成 → SciPOSの統合型と直接競合。ただしビュッフェ/NFC/無料コアで差別化可能。

### 3.3 ノルウェー

| 項目 | 詳細 |
|------|------|
| 財務コンプライアンス | **デジタル署名必須 + SAF-T v1.30エクスポート**（2025年1月義務化） |
| 主要デリバリー | Wolt、Foodora |
| 主要決済 | Vipps（オンライン取引の70%）、カード |
| 必要な対応 | SAF-T出力、デジタル署名、Vipps連携、ノルウェー語対応 |

### 北欧共通の連携優先度

| 連携先 | 優先度 | 対象市場 |
|--------|--------|---------|
| **Wolt API** | 🔴 最優先 | 全北欧 |
| **Vipps MobilePay** | 🔴 最優先 | 芬/丹/諾 |
| **Swish** | 🟡 次点 | 瑞典 |
| **Foodora API** | 🟡 次点 | 瑞典/諾威 |

## 4. Phase 3: DACH進出（ドイツ → オーストリア → スイス）

### 4.1 ドイツ（欧州最大の高価値市場）

| 項目 | 詳細 |
|------|------|
| 市場規模 | POS SW $2.47B（2026年）、外食市場$139B |
| レストラン数 | 〜170,000+ |
| 財務コンプライアンス | **KassenSichV: 認定TSE必須**（クラウドTSE可）、DSFinV-K出力、レシート発行義務、2025年7月ELSTER登録義務 |
| 主要デリバリー | **Lieferando**（75%シェア）、Wolt、Uber Eats |
| 主要決済 | **girocard**（1億+枚、84%非接触）、現金もまだ重要 |
| 主要競合 | Lightspeed/Gastrofix（8,000+）、orderbird/Nexi（14,000+）、SumUp、Vectron、Leaf Systems、allO、Fleksa |
| 必要な対応 | TSE認証（fiskaly等のクラウドTSE活用推奨）、DATEV会計連携、Lieferando連携、ドイツ語対応 |

**ドイツ市場でのSciPOSポジション:**
```
        フルスタック度 高
            │
  SciPOS ★ │  Lightspeed
   allO ●  │  orderbird
            │
 ───────────┼───────────
            │
  Leaf ●    │  Vectron
  Fleksa ●  │  SumUp
            │
        フルスタック度 低
 ←─ SMB ───┼─── エンタープライズ ─→
```

**差別化戦略:**
- **vs orderbird**: 無料コアPOS + ビュッフェ + NFC（orderbirdは€119+/月）
- **vs Lightspeed**: セルフオーダー統合 + シフト管理（LightspeedはKDS/キオスク弱い）
- **vs allO**: 最も類似する競合。NFC + ビュッフェ + GPSシフト管理で差別化

### 4.2 オーストリア・スイス（ドイツと同時展開推奨）

| 項目 | オーストリア | スイス |
|------|------------|--------|
| コンプライアンス | RKSV（レジ義務化あり） | **なし**（最低障壁） |
| デリバリー | Lieferando、Wolt | Uber Eats、Lieferando |
| 決済 | カード、現金 | **TWINT**（500万+ユーザー） |
| 言語 | ドイツ語（対応済み） | ドイツ語/仏語/伊語 |

## 5. Phase 4以降: その他の市場

### 5.1 英国・アイルランド

| メリット | デメリット |
|---------|----------|
| 財務認証不要（MTDのみ） | 競合が極めて強い（Toast/Square/Epos Now） |
| 英語市場で参入容易 | Tebiが2025年から英国展開開始 |
| デリバリー市場大（£14.3B） | GBP建て対応必要 |

**推奨:** DACH安定後に検討。英語対応は既にあるため追加開発少。

### 5.2 フランス・ベネルクス

| メリット | デメリット |
|---------|----------|
| 市場大（FR 175,000+レストラン） | **NF525認証必須**（最大の障壁） |
| ベルギーはGKS 2.0対応必要 | フランス語ローカライズ必須 |
| オランダは規制緩い | 地場競合強い（Zelty/L'Addition/Obypay） |

**推奨:** 長期目標。NF525認証は工数大。パートナー戦略が現実的。

### 5.3 日本（特別枠）

| メリット | デメリット |
|---------|----------|
| 成長率21.5%（世界最高クラス） | 参入障壁が極めて高い |
| 93万+レストラン | Airレジ（無料、12.1万店）が圧倒的 |
| ビュッフェ文化が強い（差別化チャンス） | 完全日本語化＋国内データホスティング必須 |
| インボイス制度対応需要 | 商習慣の壁（委員会型意思決定、長い営業サイクル） |

**推奨:** 現地パートナー/リセラー経由での展開。直接参入は非推奨。ビュッフェ・食べ放題チェーンに特化すれば差別化可能。

### 5.4 東南アジア

| メリット | デメリット |
|---------|----------|
| 成長率15.6%（高成長） | 価格感度が極めて高い |
| 規制障壁が比較的低い | StoreHub等の地場競合 |
| QRコード決済文化がSciPOSに適合 | 国ごとに言語/決済/規制が異なる |

**推奨:** 長期目標。マレーシア（英語通用+e-invoice 2026年義務化）が最初の候補。
