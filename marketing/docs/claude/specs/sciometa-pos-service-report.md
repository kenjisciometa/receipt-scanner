# SciometaPOS サービス総合報告書

> **作成日**: 2026-04-22
> **目的**: 市場調査・競合動向分析の基盤資料として、SciometaPOS エコシステム全体を網羅的に把握する

---

## 1. エグゼクティブサマリー

**SciometaPOS（SciPOS）** は、Sciometa Oy が提供するクラウドベースの飲食店向け総合管理プラットフォームである。BYOD（Bring Your Own Device）モデルを採用し、既存のタブレット端末で導入可能な点が最大の特徴。POS レジ機能を中心に、セルフオーダー、キッチンディスプレイ、シフト管理、会計連携、サブスクリプション課金まで、飲食店運営に必要な機能をワンストップで提供する。

### サービス全体像

| カテゴリ | アプリケーション数 | 主要技術 |
|---------|----------------|---------|
| コア POS（無料） | 4 アプリ | Next.js, React, Supabase |
| プレミアム注文系 | 4 アプリ | Flutter, React Native |
| バックオフィス系 | 3 アプリ | Next.js, Flutter |
| パートナー・課金系 | 2 アプリ | React, Next.js |
| インフラ | 2 サービス | Node.js, PostgreSQL |

## 2. プロダクトポートフォリオ

### 2.1 無料コアプロダクト

#### POS（Point of Sale）— メイン管理ダッシュボード
- **技術スタック**: Next.js 16 + React 19 + TypeScript + Supabase + Tailwind CSS
- **主な機能**:
  - 注文管理（作成・編集・返金・CSV エクスポート）
  - 商品管理（CRUD、カテゴリ、オプショングループ、翻訳、在庫連動）
  - 在庫管理（在庫追跡、カートン管理、アラート、入出庫、インポート/エクスポート）
  - テーブルマップエディタ（ビジュアルレイアウト設計）
  - 売上レポート・分析（KPI ダッシュボード）
  - マルチ店舗管理
  - RBAC（ロールベースアクセス制御: owner / admin / manager / cashier）
  - 12+ 言語対応（next-intl）
  - 決済端末連携（SumUp、Nets）
  - レシート印刷（ESC/POS サーマルプリンタ）
  - 会計ソフト連携（Netvisor、Procountor）
  - AI チャットボット（OpenAI SDK）
  - NFC タグ / QR コード / ESL（電子棚札）連携
- **URL**: pos.sciometa.com
- **ポート**: 9001（dev）、80/443（production via Docker）

#### KDS（Kitchen Display System）
- **技術**: React Native（本番） / Flutter（レガシー）
- **機能**: リアルタイム注文表示、3段階ステータス管理（pending → ready → served）、カテゴリ別ルーティング、複数ディスプレイ対応

#### CDS（Customer Display System）
- **機能**: 注文内容・金額のリアルタイム顧客向け表示

#### SDS（Server Display System）
- **機能**: 注文完了通知、スタッフへの配膳指示

### 2.2 プレミアムプロダクト（有料）

#### BOS（Bell Order System）— セルフオーダーキオスク
- **技術**: React Native + Expo
- **価格**: €90/月（店舗単位）
- **主な機能**:
  - タッチスクリーン対応セルフ注文
  - ビジュアルメニュー表示（写真・説明・価格）
  - カスタマイズ注文（モディファイア）
  - 複数決済対応（SumUp、Nets、現金）
  - キオスクモード（ロック画面）
  - 注文番号発行、QR コードレシート
  - 12+ 言語対応
  - KDS 直接連携

#### TOS（Table Order System）— テーブルオーダー
- **技術**: Flutter / タブレット最適化
- **価格**: €90/月（店舗単位）
- **主な機能**:
  - QR コードによるセッション開始
  - テーブルからの直接注文
  - 追加注文対応（1回の食事で複数注文）
  - リアルタイムステータス追跡
  - 横向き最適化 UI
  - 特別リクエスト（メモ付き注文）

#### Self Station — セルフチェックアウト
- **技術**: Flutter（iOS / Android / Web / Linux）
- **価格**: €90/月（店舗単位）
- **主な機能**:
  - ビジュアルテーブルマップ表示
  - セッションコードシステム（A001〜Z999）
  - **割り勘（スプリットビル）**:
    - 均等割り / アイテム別割り当て
    - 参加者ごとの個別決済
    - 複数カード対応
    - オプティミスティック UI 更新
  - ビュッフェダインイン注文フロー
  - バーコード/QR レシート印刷
  - 無操作検知・自動リセット
  - キオスクロックダウン（モバイル/デスクトップ）
  - 12言語対応

#### OSD（Order Status Display）— 注文状況表示
- **機能**: 顧客向け注文準備状況のリアルタイム表示
- **効果**: スタッフの呼び出し作業を 50% 削減

#### QR Menu — デジタルメニュー
- **機能**: QR コードでアクセスするウェブメニュー

#### Reservation — 予約管理
- **機能**: テーブル予約、予約 URL 生成、分析機能

  - **支出タブ**: 経費（レシートスキャン）、請求書管理
  - **労務タブ**: 労務コスト分析、タイムシート
  - 税率別内訳レポート
  - 割引分析
  - CSV エクスポート（欧州フォーマット: セミコロン区切り、カンマ小数点）
  - 日付範囲フィルタ（プリセット + カスタム）
  - 店舗フィルタ（マルチセレクト）
  - タイムゾーン対応
  - Account ID によるセキュアアクセス（ACC-XXXXX 形式）
  - Gmail 連携（請求書/レシート自動スキャン）
  - Procountor 連携
- **URL**: tax.sciometa.com

#### Receipt Scanner — レシートスキャナー
- **技術**: Flutter（モバイル） + Next.js（Web）

  - AI OCR（Google ML Kit Text Recognition）

  - データ抽出・検証

  - CSV/JSON エクスポート
  - 画像前処理（角度補正、明るさ調整、遠近補正）

### 2.4 プラットフォーム・インフラ

#### Subscription Web — サブスクリプション管理サイト
- **技術**: Next.js 16 + Stripe + Supabase
- **主な機能**:
  - 3 ティア料金体系の表示・比較
  - Stripe Checkout 連携
  - トライアル資格管理（14 日間無料、クロスティアブロッキング）
  - サブスクリプションライフサイクル管理（開始 → トライアル → 有料 → 変更 → 解約）
  - ティア変更プレビュー（アップグレード / ダウングレード）
  - 月額 ↔ 年額切替
  - Google Play / Apple 決済対応
  - OAuth ログイン（Google、Apple）
- **URL**: subscription.sciometa.com

#### Partner App — パートナープログラム管理
- **技術**: React 18 + TypeScript + Vite + MUI + Redux Toolkit
- **主な機能**:
  - パートナー登録・承認ワークフロー
  - アクティベーションコード生成・追跡
  - コミッション管理（初回 80% + 36 ヶ月間継続 10%）
  - Stripe Connect 自動ペイアウト
  - 地域別料金対応（北欧、中央/西欧、東欧）
  - 収益・顧客分析ダッシュボード
  - 管理者パネル

#### WebSocket Server — リアルタイム通信サーバー
- **技術**: Node.js + Express + Socket.IO + PostgreSQL LISTEN/NOTIFY + Redis
- **目的**: Supabase Realtime の代替として高性能なリアルタイム通知を提供
- **主な機能**:
  - PostgreSQL トリガーベースの通知（pg_notify）
  - 数千同時接続対応のスケーラブル設計
  - JWT デバイス認証
  - Redis Streams による信頼性の高いメッセージ配信
  - ヘルスチェック・メトリクス・監視
  - 複数デバイスタイプ対応（KDS、CDS、SDS）

## 7. 差別化要因・バリュープロポジション

### 7.1 コスト比較

| 項目 | 従来型 POS | SciPOS |
|-----|-----------|--------|

## 10. 競合分析のための整理

### 10.1 SciPOS の位置づけ

**カテゴリ**: クラウドベース・タブレット POS / 飲食店向け SaaS
**セグメント**: SMB（小〜中規模飲食店）向けフリーミアム型

### 10.2 比較対象となる競合カテゴリ

| カテゴリ | 競合例 | SciPOS との差別化ポイント |
|---------|-------|----------------------|
| **タブレット POS** | Square POS, Toast, Lightspeed | BYOD・無料コア・12言語対応 |
| **セルフオーダーキオスク** | Aloha, Kiosk, QikServe | タブレット利用で低コスト導入 |
| **飲食店 SaaS** | MarketMan, 7shifts, HotSchedules | POS 統合型ワンストップ提供 |
| **クラウド POS** | iZettle, SumUp POS, Zettle | 会計連携・キッチンディスプレイ統合 |
| **テーブルオーダー** | me&u, Mr Yum, Bbot | NFC タップ注文、割り勘機能 |

### 10.3 市場調査で確認すべきポイント

1. **価格競争力**: 競合の店舗単位月額との比較（€90/店舗は競争力があるか）
2. **BYOD モデルの市場受容性**: ハードウェアバンドル vs. BYOD の好み
3. **多言語対応のニーズ**: EU 圏での多言語 POS への需要度
4. **会計連携**: Netvisor / Procountor 以外の需要（Xero, QuickBooks 等）
5. **フリーミアムモデルの有効性**: 無料 POS → 有料アップセルの転換率ベンチマーク
6. **パートナープログラム**: 競合のチャネルパートナー戦略との比較
7. **セルフオーダー市場の成長率**: COVID 後の非接触注文への移行トレンド
8. **AI 活用**: 競合の AI 機能（需要予測、自動発注等）の導入状況

## 12. まとめ

SciometaPOS は **「BYOD + フリーミアム + ワンストップ」** の 3 つの柱で、飲食店向け POS 市場に独自のポジションを確立しようとしている。コア POS を無料で提供しつつ、セルフオーダー（BOS/TOS/Self Station）、シフト管理、会計連携といったプレミアム機能で収益化するモデルは、導入障壁を下げ、ロックインを回避するという SMB 市場のニーズに合致している。

12+ 言語対応、組織タイムゾーン対応、地域別料金設定により、EU 圏全域への展開基盤が整っている。また、AI を活用した自動発注システムの開発は、将来的な差別化要因として期待できる。

市場調査においては、特に **BYOD モデルの受容性**、**フリーミアム転換率**、**セルフオーダー市場の成長性**、**競合の AI 活用状況** に注目すべきである。
