## ハードウェアモデル

- **専用端末**: Hypersoft 推奨の業務用タッチスクリーン端末（カウンタ型・モバイル型）
- **BYOD**: 部分対応（タブレット利用可能だが、推奨HWあり）
- **対応決済端末**: Hypersoft Pay、Adyen モバイル端末。Visa/Mastercard/非接触対応
- **プリンタ対応**: 業務用レシートプリンタ・キッチンプリンタ対応
- **SOT（セルフオーダーキオスク）**: Hypersoft 専用タッチスクリーンキオスク端末
- **KDS モニター**: 専用キッチンディスプレイ
- **周辺機器**: カスタマーディスプレイ、マウント金具、モバイル決済端末

## 強み / 弱み（SciometaPOS との比較視点）

### 強み
- **ホテルF&B 連携の深さ**: 15以上のホテルPMS（Oracle、Mews、Protel 等）と直接統合。ホテルのレストラン・バーでは他POS製品に勝る強み
- **Deliverect 経由のデリバリー網**: Lieferando をはじめとする主要デリバリープラットフォームへの統合が確立
- **DATEV / 会計統合の完全性**: DATEV、ABACUS、SAP HR まで対応する広範な会計連携
- **SOT（キオスク）のネイティブ統合**: Self Order Terminal が Hypersoft eSolutions に完全統合されており、QRオーダーとキオスクの両立が可能
- **KDS の二系統**: QSR向けと通常レストラン向けに異なるKDS製品を用意
- **Volaris Group（Constellation Software 傘下）の後ろ盾**: 安定した長期経営。Volaris はソフトウェア企業を解体せず長期保有する方針

### 弱み
- **価格の不透明性**: 完全カスタム料金で、オンラインでの比較・検討が困難
- **Jamezz 連携が未完成**: 公式ドキュメントによると Jamezz 統合は「計画段階・技術未実装」と明記されており、マーケティング上での訴求と実態に乖離
- **モジュール追加コストが高い**: 在庫管理・モバイル端末等の機能追加が都度課金となり、総コストが膨らむとのユーザー指摘
- **SME 向けではない**: 複雑なシステム構成と高いエントリーコストから、小規模店への普及は限定的
- **シフト管理の統合が浅い**: 勤怠データの外部エクスポートは可能だが、SciometaPOSのようなシフト管理統合はない
- **従業員規模が小さい**: 約47名のエンジニアリングチームでは、急速な機能開発に制約がある可能性

### SciometaPOS との差異ポイント
- SciometaPOSはPOS＋セルフオーダー＋シフト管理の一体型提供が強み。Hypersoftはシフト管理が外部依存
- Hypersoftのホテル連携・DATEV統合はエンタープライズ用途での強みだが、日本・アジア市場には無関係
- Jamezz連携（SciometaPOSの競合であるJamezz）はHypersoftが「Jamezz対応」と宣伝しているが実装が未完成という点は、競合分析上の重要事実
