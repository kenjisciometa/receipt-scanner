## ハードウェアモデル

- **BYOD対応**: iOS・Android・Windows・macOS 端末で動作（既存タブレット・スマートフォン・PC使用可能）
- **専用端末**: ready2order 独自ハードウェア（readyMini / readyGo / readyTab / readyMax 2）も販売
- **対応決済端末**: readyPay（内蔵、1.23%手数料）。Visa/Mastercard/ApplePay/GooglePay 対応
- **プリンタ対応**: 主要レシートプリンタ対応
- **ハードウェア特徴**: readyGo・readyMax はオールインワン設計でPOSとカード端末が一体化

## 強み / 弱み（SciometaPOS との比較視点）

### 強み
- **DACH 特化の法規制完全対応**: TSE（ドイツ）・RKSV（オーストリア）・GoBD に完全準拠。自動更新で常に最新規制に対応
- **低価格・シンプルな料金体系**: €23〜35/月からで小規模店でも導入しやすい。モジュール制で必要な機能だけ課金
- **マルチプラットフォーム（BYOD）**: iOS/Android/Windows/macOS 対応で既存端末を活用可能。ハードウェアコストが低い
- **Zucchetti グループの後ろ盾**: イタリア大手IT（9,000名以上）の傘下でリソース・技術力が強化
- **SME 向けの使いやすさ**: クイックセットアップ、日次8時〜22時のサポート、AIアシスタント
- **DATEV/BMD 直接エクスポート**: ドイツ・オーストリアの税理士とのワークフローにシームレスに統合

### 弱み
- **エンタープライズ機能の不足**: 大規模チェーン向けの高度な機能（複数店舗管理、高度分析）が弱い
- **セルフオーダー・KDS がネイティブ非対応**: サードパーティ連携が必要で、SciometaPOS の統合体験に劣る
- **シフト管理の統合が浅い**: 勤怠管理・シフトスケジューリングは外部ツール依存
- **デリバリー連携の弱さ**: Lieferando 等との直接ネイティブ連携が非公表
- **DACH 圏外での展開なし**: グローバル展開に向いていない

### SciometaPOS との差異ポイント
- SciometaPOSはセルフオーダー・シフト管理のネイティブ統合が強み。ready2orderはこれらをサードパーティ依存
- ready2orderの低価格・シンプルさはSME向けに強力だが、中規模以上の飲食店運営者にとってはSciometaPOSの統合深度の方が価値が高い
- 地域的に競合しない（ready2orderはDACH特化、SciometaPOSは日本・アジア）
