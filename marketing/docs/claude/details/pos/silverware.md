## 強み / 弱み（SciometaPOS との比較視点）

### 強み
- **ホテルPMS連携の深さ**: Opera、Maestro等との双方向リアルタイム連携は業界トップクラス。ゲストプロファイル・フォリオ・ロイヤルティデータの統一
- **エンタープライズ対応力**: 82プロパティの大規模ロールアウト実績。マルチベニュー・マルチプロパティ管理
- **オフライン耐性**: ローカルSQLエンジン + クラウド同期のハイブリッドアーキテクチャ
- **モジュラー設計**: 必要な機能のみ選択的にデプロイ可能
- **30年の実績**: ホスピタリティ業界への深い理解とスタッフの長期在籍
- **ゲスト体験**: GuestXによるアプリ不要のモバイル注文・決済
- **導入支援**: ホワイトグローブ実装サービス

### 弱み
- **料金非透明**: 完全カスタム見積もり。SMBには敷居が高い
- **個人店向けではない**: エンタープライズ・ホテル特化。小規模レストランには過剰
- **在庫・シフト管理は統合依存**: 自社機能は限定的。Push Operations等サードパーティ連携が必要
- **地域制限**: カナダ・北米中心。グローバル展開は限定的
- **多言語制限**: アジア言語非対応
- **Fullsteam傘下**: PEロールアップによる今後の戦略変更リスク

### SciometaPOS との比較
- Silverwareのホテル・リゾート向けPMS連携はSciometaPOSにはない専門性
- SciometaPOSはシフト管理を自社統合 → Silverwareはサードパーティ依存
- SciometaPOSはアジア市場・多言語対応 → Silverwareは北米中心
- SciometaPOSはSMB〜中規模を主対象 → Silverwareはエンタープライズ特化
- Silverwareのオフライン耐性（ローカルSQL）はSciometaPOSにとって参考になる技術アプローチ
