## 料金体系

### 月額プラン（USドル建て公式サイト掲載価格 / ロケーション・月間注文数ベース）

| プラン | 月額 | 月間注文数上限 | 初期費用（セットアップ） |
|--------|------|--------------|----------------------|
| Small | $119 / €119 | 350件以下 | $249 / €249 |
| Medium | $199 / €199 | 351〜750件 | $249 / €249 |
| Large | $249 / €249 | 751〜1,500件 | $249 / €249 |
| Enterprise | 要問い合わせ（カスタム） | 1,500件超 | 要問い合わせ |

- 上限超過分はオーバーチャージが発生
- **課金単位**: ロケーション（店舗）単位 × 注文量ティア
- **無料プラン / トライアル**: なし（デモのみ）
- **年間契約**: 原則年間契約。一部ユーザーレビューでは90日前キャンセル通知が必要との報告あり
- **Deliverect Kiosk（Tabesto）**: 別途見積もり。ハードウェア費用は公表なし

## 機能比較

| 機能カテゴリ | 詳細 | 対応状況 |
|------------|------|---------|
| **デリバリー注文統合（コア機能）** | Uber Eats・DoorDash・Deliveroo・Just Eat・Grubhub等 1,000+ チャネルを POS へ自動注入 | ◎ 業界最強クラス |
| **POS 連携（注文インジェクション）** | Toast・Square・Lightspeed・Clover・TCPOS・Aloha Cloud 等 500+ POS と統合 | ◎ |
| **セルフオーダー（キオスク）** | Tabesto買収（2024年12月）により自社キオスクハードウェアを取得。カウンタートップ型・スタンドアロン型。英・仏・独・西・伊・ベルギー・スイスで展開 | ◎ |
| **セルフオーダー（QR / テーブル）** | 統合パートナー経由（直接提供は薄い） | △ |
| **KDS（キッチンディスプレイ）** | KDS パートナー統合経由 | ◯（統合） |
| **在庫管理** | POS からのリアルタイム在庫同期、CSV インポート、品切れ時の自動非表示。専用在庫ソフトとの統合カテゴリあり | ◯ |
| **シフト・勤怠管理** | 自社機能なし。パートナー統合経由 | △（統合のみ） |

| **CRM / ロイヤルティ** | Thanx・Paytronix 等との統合。Deliverect AI が注文データを活用したパーソナライズドマーケティングをサポート | ◯（統合） |
| **デリバリー連携** | ◎ コア機能。1,000+ 統合 | ◎ |
| **会計ソフト連携** | 統合エコシステム経由 | ◯ |
| **レポート・分析** | Deliverect Pulse：マーケットプレイスの売上・競合比較・ストアヘルス分析。多ロケーション対応 | ◎ |
| **Pulse Sentinel** | ストア稼働状況のリアルタイム監視・自動再開。バーガーキングUKで月4,000件のダウンタイム防止実績 | ◎ |

## 強み / 弱み（SciometaPOS との比較視点）

### 強み

| 強み | 詳細 |
|------|------|
| デリバリー統合の圧倒的規模 | 1,000+ 統合、累計 10 億注文超、世界 78 カ国対応。競合が追いつけないネットワーク効果 |
| エンタープライズ顧客基盤 | KFC・バーガーキング・タコベル・パパジョンズ・リトルシーザーズ等の超大手と深い関係 |
| ユニコーン評価・財務基盤 | $332M+ 調達済み。次世代 AI / ハードウェア投資余力が大きい |
| AI エージェント（2026年） | 自律型メニュー最適化・マーケティング自動化は業界最先端。KFC での実績あり |
| キオスク（Tabesto 統合） | デリバリー × キオスクの融合。オムニチャネル完結に向かっている |
| グローバルカバレッジ | 52〜78 カ国対応。MENA 急成長。アジア展開も進行中 |
| 高可用性 | 99.998% アップタイムは実運用でも証明済み |

### 弱み

| 弱み | 詳細 |
|------|------|
| 本来はデリバリー統合特化 | 自社 POS 機能なし。POS は統合先に依存 |
| 予約・シフト管理機能なし | レストランOS としての完全性は欠ける |
| 料金体系が注文量依存 | 注文が多い店舗ほどコスト増。大量注文店には割高 |
| セットアップの複雑さ | ユーザーレビューでオンボーディングの難しさを指摘する声が多数 |
| 年間契約縛り | 90 日前キャンセル通知などの契約条件でクレームあり |
| カスタマーサポート評価 | 一部ユーザーから対応の遅さが指摘されている |
| アジア市場での実績薄 | 日本・東南アジア向けのローカライズや営業体制は未整備 |
| QR テーブルオーダー弱い | インストア QR 注文は自社製品でなくパートナー統合依存 |

### SciometaPOS との差異

| 比較軸 | Deliverect | SciometaPOS |
|--------|-----------|-------------|
| コアバリュー | デリバリー統合・オムニチャネル統合 | POS + セルフオーダー + シフト管理の垂直統合 |
| ターゲット市場 | 欧米大手チェーン・エンタープライズ | 日本・アジアのレストラン（個人店〜チェーン） |
| POS 機能 | なし（統合のみ） | あり（自社開発） |
| シフト管理 | なし | あり（コア機能） |
| セルフオーダー | キオスク（Tabesto）中心 | QR テーブルオーダー + キオスク |
| デリバリー統合 | ◎ 圧倒的強み | 基本的な統合のみ |
| 価格帯 | 中〜高（企業向け） | 中（アジア市場向け） |
| 競合関係 | 直接競合は薄い（異なるセグメント） | デリバリー統合機能の補完的関係も考えられる |
