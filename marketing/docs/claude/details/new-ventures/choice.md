## 料金体系

### 月額プラン（ユーロ建て・公式サイト掲載価格）

| プラン | 月額（目安） | 主な位置づけ |
|--------|------------|------------|
| Basic | €13〜15 / 月 | QR メニュー表示のみのエントリープラン |
| Standard | €25.5〜30 / 月 | オンライン注文・決済追加 |
| Smart | €51〜60 / 月 | 予約・ロイヤルティ・マーケットプレイス連携 |
| Pro | €75〜89 / 月 | フル機能 + AI 機能 + 高度な分析 |

- **課金単位**: 店舗（ロケーション）単位
- **無料プラン / トライアル**: 無料トライアルあり（クレジットカード不要）
- **初期費用**: なし（または低額）
- **年間 ARR**: $5M+ 達成（2026年初頭時点）
- **月間 GMV**: 約 €35M（2026年時点）

> 注：価格帯は複数のレビューサイト・メディア記事からの推定値。公式サイトの正確な現行料金は choiceqr.com/pricing にて確認要

## 強み / 弱み（SciometaPOS との比較視点）

### 強み

| 強み | 詳細 |
|------|------|
| 圧倒的な低コスト・シンプルさ | 月額 €13〜89 の幅広い価格帯。小規模店でも導入しやすい。BYOD なので初期費用が最小 |
| オールインワン SaaS（単一サブスクリプション） | QR メニュー・決済・テイクアウト・マーケットプレイス・予約・ロイヤルティを 1 プランで提供 |
| コミッションフリー注文 | マーケットプレイス手数料（15〜30%）を回避できる自社チャネルは独立系オーナーに強い訴求 |
| 顧客データ所有権 | 全チャネルの顧客データをレストランが直接保有・活用できる（マーケットプレイス依存からの脱却） |
| 急成長・資本効率 | $11.6M という少額調達で 30,000+ 店舗・月次黒字を達成。SaaS としての効率の高さ |
| CEE 市場での深い根付き | チェコ・ポーランド・ハンガリー等、まだ大手が本格参入していない市場でのローカル優位性 |
| 南欧拡大中 | ポルトガル・スペイン・イタリアへの展開開始。EU 全域でのスケールポテンシャル |

### 弱み

| 弱み | 詳細 |
|------|------|
| 資金力の小ささ | $11.6M は Deliverect の $332M と比べると 28 倍以上の差。大手との競合では資金力が課題 |
| 自社 POS 機能なし | レジ機能を持たず、POS は他社連携に依存。「OS」とは言いつつ完全ではない |
| シフト・勤怠管理なし | スタッフ管理機能は未提供。レストラン経営の全体最適には対応不足 |
| キオスク端末なし | QSR 向けのセルフオーダーキオスクハードウェアは提供していない |
| AI 機能は開発中 | 2026年3月の Series A 資金で開発中。競合（Deliverect）との差が開くリスク |
| エンタープライズ対応限定 | 大手チェーン向けの多拠点管理・高度なレポート等の機能は不十分 |
| ウクライナリスク | 展開市場にウクライナを含む。地政学リスクあり |
| 西欧・北米は未参入 | ドイツ・フランス・UK・北米は次フェーズ。競合が強い市場での戦いはこれから |
| 日本・アジア展開なし | 東アジア市場は対象外。言語・決済・規制の複雑さを考慮すると短期参入は考えにくい |

### SciometaPOS との差異

| 比較軸 | Choice（ChoiceQR） | SciometaPOS |
|--------|------------------|-------------|
| コアバリュー | QR 注文 + コミッションフリー + 顧客データ統合 | POS + セルフオーダー + シフト管理の垂直統合 |
| ターゲット市場 | 欧州（CEE → 南欧）の独立系・中小規模レストラン | 日本・アジアのレストラン（個人店〜チェーン） |
| POS 機能 | なし（外部連携） | あり（自社開発） |
| シフト管理 | なし | あり（コア機能） |
| セルフオーダー | QR テーブルオーダー中心 | QR テーブルオーダー + キオスク |
| デリバリー統合 | マーケットプレイス連携あり（コア機能のひとつ） | 基本的な統合 |
| 価格帯 | 低〜中（月額 €13〜89） | 中（アジア市場向け） |
| AI 機能 | 開発中 | 開発中 |
| 競合関係 | 直接競合は薄い（異なる地域・機能）。QR 注文・顧客データ統合のアプローチは参考になる |
