### データ連携システム一覧

Toast / NCR（Aloha） / Square / Restaurant365 / 7shifts / Olo

### AIの具体的な内容

- **「James」AIエージェント**: チャット・テキスト・電話でQSR運営データに自然言語でクエリできる会話型AIアシスタント。「先週の最も収益性の高い店舗はどこか」「火曜のランチ時間帯のUPTが下がった理由は何か」などを口語で質問可能
- **自動データ統合**: POS・労務・在庫・会計の複数システムからデータをリアルタイム集約。Rogersは「週13回のCSVエクスポート」を廃止
- **自動レポート**: 日次・週次の経営サマリーをスラック/SMS/メール経由で自動配信
- **アクション実行（開発中）**: 価格調整・シフト再スケジュール・プロモーション実行のエージェント型自動化。現在は「インサイト提供→人間が承認→実行」だが、将来的には完全自動を目標
- **マルチチャネル対応**: テキスト・通話・チャット・任意のチャンネルで「いつでも」データにアクセス可能

## 強み / 弱み（SciPOSとの比較視点）

### 強み（Claveが優れる点）

1. **フランチャイズデータ統合の深さ**: 複数ブランド・複数POS・複数システムを一元統合できる点は、フランチャイジーにとって大きな価値。SciPOSは自社POS内データに閉じているが、ClaveはサードパーティPOSとの横断比較が可能
2. **自然言語インターフェース**: データアナリスト不要で、オーナー・マネージャーが口語でビジネスデータを照会できる。レポート作成の民主化
3. **実業出身の創業者**: ChurroManiaの実運営経験から生まれた課題設定が現場に刺さる。QSRフランチャイジーの信頼獲得が早い
4. **TechCrunch Disrupt 2025登壇**: 業界認知・採用・調達における信頼性向上
5. **高い粗利率（約60%）**: スケール段階でも経済的に持続可能なモデル

### 弱み（SciPOSが優れる点）

1. **POSなし・店内オペレーション管理なし**: POS・セルフオーダー・KDS・在庫管理・シフト管理などオペレーション機能は一切提供しない。SciPOSは統合型
2. **分析ツールとしてのみ価値を持つ段階**: 現状は「インサイト提供」に留まり、自律的なアクション実行はまだ限定的（開発中）
3. **極初期スタートアップ**: Pre-Seed $850K・6社のみ。安定性・スケーラビリティに懸念あり
4. **QSR以外には対応しない**: フルサービスレストラン・カフェ・ホテルF&Bなど他業態には不向き
5. **顧客獲得コストが高い可能性**: LinkedInのDM戦略から始まった草の根展開は初期段階では有効だがスケールに課題
6. **競合脅威**: Toast・Olo・Restaurant365などの大手POS・管理システムが分析機能を自社で強化する可能性あり

## 調査メモ

- ClaveはPOSやオーダーシステムではなく、**フランチャイズ向けのAI分析・意思決定自動化レイヤー**として位置づけるべき。SciPOSとの競合は少なく、むしろSciPOSと連携するポジションに近い
- 「James」というAIエージェント名は現在開発中。完成すれば業界初の「QSRオペレーション全自動エージェント」になる可能性があるが、現在は主にインサイト提供段階
- $850K調達は極めて少額。今後のシリーズAが実現するかどうかで生存可能性が大きく左右される
- 創業者の家族事業（ChurroMania）が最初の大顧客であるため、客観的な成功事例としての信頼性に留意が必要
- SciPOSがフランチャイズ向けに横断データ分析・レポーティング機能を強化すれば、このセグメントでの差別化になりうる
