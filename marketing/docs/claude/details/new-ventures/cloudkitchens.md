## ビジネスモデル

### 3層構造
1. **不動産層**: 元商業施設・倉庫を改装したゴーストキッチン施設を運営（テナントにレンタル）
2. **ソフトウェア層**: OtterによるPOS・注文管理・分析ツール
3. **フードデリバリー層**: Future Foods（バーチャルブランド）でUber Eats・DoorDash等への配信

### 収益構造
| 収益源 | 内容 |
|--------|------|
| キッチンレンタル | 初期費用$30K相当＋月額$3,500〜$6,000/ユニット |
| Otter SaaS | $19〜$59/月（POS）、$99〜$149/月（注文アグリゲーション） |
| Future Foods | 売上の約10%レベニューシェア |

## Otter（POSソフトウェア）の機能

| 機能 | 詳細 |
|------|------|
| 注文アグリゲーション | Uber Eats・DoorDash・Grubhub 等を一元管理 |
| POS端末 | OtterPOS（Stripeペイメント連携） |
| KDS（キッチン表示） | 注文をキッチンディスプレイに自動転送 |
| セルフオーダーキオスク | 店内キオスク端末対応 |
| 在庫管理 | 基本的な在庫追跡 |
| 売上分析 | Revenue Center Analytics（2025年）：チャネル別手数料・返金率・調理時間比較 |
| メニュー最適化 | 複数プラットフォームのメニュー一元管理 |
| オフライン対応 | OtterPOS はオフライン時も継続稼働 |

### Otter の市場規模
- 米国フードデリバリー注文の18%を処理（2024年）
- 275,000以上のユーザー
- 100,000以上のレストランが利用

## 強み（SciometaPOS との比較視点）

1. **不動産＋ソフトウェアの垂直統合**: 競合が容易に模倣できないユニークなバリューチェーン
2. **Otterの注文アグリゲーション**: 米国デリバリー注文18%処理は圧倒的な規模
3. **エンタープライズ顧客基盤**: Chick-fil-A等の大手チェーンとの取引実績
4. **$15Bバリュエーション**: 豊富な資金力でグローバル展開加速
5. **Travis Kalanick のブランド力**: スタートアップ・投資家コミュニティでの知名度
6. **AI調理コンセプト**: 将来的な「Cooking as a Service」「AI最適化料理」ビジョン

## 弱み（SciometaPOS との比較視点）

1. **テナント離脱率が高い**: 2022年データで65%が1年以内に退去
2. **ゴーストキッチンモデルの限界**: 実店舗を持つレストランへの普及に障壁
3. **北米中心**: 欧州・アジア市場では存在感が薄い
4. **シフト管理・勤怠機能なし**: OtterはHRツールを持たない
5. **中東IPO延期**: 成長ストーリーへの疑問符
6. **競合激化**: Deliverect、Olo、Toast等との注文管理市場での競争
7. **POS機能は限定的**: シフト管理・多言語・NFC等でSciometaPOSに劣る

## SciometaPOS との競合・協業関係

| 区分 | 評価 |
|------|------|
| 直接競合（POS） | **中〜高**（Otterのキオスク・KDS・POS機能はSciometaPOSと重複） |
