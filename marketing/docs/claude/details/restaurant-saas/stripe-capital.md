## 機能比較

| 機能 | 対応状況 |
|------|----------|
| データドリブン審査 | ○ Stripe決済データ（処理量、成長軌道、チャージバック率、顧客基盤規模）をアルゴリズムで分析 |
| POS/決済データ連携 | ○ Stripeの処理データをリアルタイムで活用。非Stripe決済データのインポートも可能 |
| リアルタイムリスク評価 | ○ 継続的なモニタリングと定期的な再評価 |
| プラットフォーム/ホワイトラベル | ○ Stripe Connect経由。3段階：ノーコード、埋め込みUI、フルAPI。プラットフォームがブランディング可能 |
| マーチャント向けレポート | ○ Stripeダッシュボードで返済状況・残高を確認可能 |
| 外部データインポート | ○ 非Stripe決済データをアンダーライティングにインポート可能 |

## 強み（SciPOSとの比較視点）

- **圧倒的なスケール**: $1.9Tの年間処理額、$9B+の累計融資実績
- **ホワイトラベル対応**: プラットフォーム（SaaS POS等）が自社ブランドで資金提供を組み込める
- **3段階の統合レベル**: ノーコードからフルAPIまで、プラットフォームの技術力に応じた統合が可能
- **手数料の透明性**: 固定フラットフィーのみ、隠れたコストなし
- **グローバル展開**: 欧米5カ国+豪州で利用可能
- **迅速な資金提供**: 1〜2営業日で着金

## 弱み（SciPOSとの比較視点）

- **レストラン特化ではない**: 汎用決済プラットフォームであり、レストラン固有のデータ（テーブル回転率、メニューミックス、在庫等）は活用しない
- **オンライン決済中心**: 対面POS機能は限定的（Stripe Terminalはあるが、専用POS端末ではない）
- **招待制**: 全ユーザーが利用できるわけではなく、Stripeが選定した対象者にオファーを送付
- **日本未対応**: 日本でのStripe Capital展開はなし
- **POS統合の深さに欠ける**: レストラン経営の全体像（労務、在庫、予約等）を把握する仕組みはない

## 最近の動向（直近1年）

- 2025年5月: AIベースの決済基盤モデルとステーブルコイン対応アカウントを発表
- 2025年2月: Bridge社（ステーブルコインプラットフォーム）を$1.1Bで買収
- 2026年2月: テンダーオファーで$159B評価額（前年$91.5Bから70%増）
- Stripe Capitalの累計融資額は$9B超に到達（2024年9月時点）
- プラットフォーム向けCapital機能の拡充（埋め込みUI、外部データインポート等）

## 調査メモ

- Stripe CapitalはSciPOSにとって「連携先」としての可能性が高い。Stripe Connectプラットフォームとして統合し、加盟店にCapital機能を提供することが技術的に可能
- ただし、Stripeは日本でのCapital展開が未定であり、短期的な連携は現実的でない可能性
- SciPOSが持つレストラン固有データ（テーブル回転率、メニュー分析、在庫、シフト管理）は、Stripeの汎用決済データでは得られない独自の審査材料になりうる
- ホワイトラベルモデルは、SciPOSが独自のエンベデッドファイナンスを構築する際の参考アーキテクチャとなる
