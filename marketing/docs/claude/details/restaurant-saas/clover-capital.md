## 料金体系（Clover Capital）

| 項目 | 内容 |
|------|------|
| 金融商品名 | Clover Capital |
| 商品タイプ | マーチャント・キャッシュ・アドバンス（MCA） |
| 提供額範囲 | 報道・レビューサイトによると$1,500〜$500,000程度（公式未公表） |
| 手数料構造 | **ファクターレート方式**。報道によるとファクターレートは1.1〜1.5程度（例：$10,000のアドバンスに対し$11,000〜$15,000を返済）。公式な料率は非公開 |
| 返済モデル | Clover POS経由の日次売上から一定割合を自動天引き。売上連動型で固定スケジュールなし |
| 審査基準 | Clover POSの売上データに基づく。最低処理期間・最低処理量の要件あり（具体的な閾値は非公開）。事業者が招待制でオファーを受ける |
| 資金着金 | 報道によると承認後1〜3営業日 |
| 申請プロセス | Cloverダッシュボードに適格者向けオファーが表示される。ワンクリックで申請可能 |

### 提供パートナー

Clover Capitalの実際の資金提供は、Fiserv/First Data傘下またはサードパーティの貸付パートナーを通じて行われる。

## 機能比較

| 機能 | 対応状況 |
|------|----------|
| データドリブン審査 | ○ Clover POSの売上データを活用したアルゴリズム審査 |

## 強み（SciPOSとの比較視点）

- **POS一体型**: 決済処理とPOS端末が一体化しているため、売上データの精度が非常に高い
- **SMBフォーカス**: 中小飲食店の利用が多く、レストラン業界での認知度が高い
- **シームレスなUX**: POSダッシュボードからワンクリックで資金調達。事業者の手間が最小限
- **Fiservの信頼性**: 親会社は$20.5Bの売上を持つフィンテック大手。資金力・規制対応力が強い
- **大規模な加盟店基盤**: 600万以上のマーチャントロケーション（Fiserv全体）
- **日次売上データ**: POS端末からのリアルタイム売上データにより、銀行よりも精緻な審査が可能

## 弱み（SciPOSとの比較視点）

- **米国限定**: 国際展開が限定的。欧州・アジアではClover Capitalは利用不可
- **ファクターレートの不透明性**: 具体的な料率が公開されておらず、比較が困難
- **レストラン特化機能の不足**: POSは汎用型であり、レストラン固有の経営分析（メニューエンジニアリング、食材原価率等）は限定的
- **ホワイトラベル制限**: 外部SaaSプラットフォームが自社ブランドでClover Capitalを提供することは困難
- **Fiserv依存**: 親会社の戦略変更により、Capital機能の方針が変わるリスク
- **手数料が高い可能性**: MCAのファクターレートは一般的にローンよりもコストが高い（年率換算で20〜80%相当になることがある）
- **2025年の課題**: Cloverプラットフォームの手数料に関する顧客苦情が報道され、株価に影響

## 最近の動向（直近1年）

- 2025年10月: Cloverプラットフォームの手数料に関する顧客苦情が報道され、Fiserv株価に影響

## 調査メモ

- Clover CapitalはSciPOSにとって最も直接的な競合モデル。「POS端末 + エンベデッドファイナンス」の組み合わせ
- ただしCloverは汎用POS、SciPOSはレストラン特化。SciPOSが持つレストラン固有データ（テーブル回転率、食材原価率、メニューミックス、予約データ、シフト管理等）は、Cloverの売上データだけでは得られない審査材料
- Clover Capitalの具体的な料率・条件が非公開であるため、ベンチマークとしての精度には限界がある
- MCAモデル（ファクターレート方式）は実質的なコストが高い傾向があり、SciPOSが金融商品を設計する際はより透明性の高い固定フラットフィー（Stripe/Adyenモデル）の方がマーチャント受けが良い可能性
- Fiserv規模の資金力がないと独自のMCAプログラムは難しいが、パートナーシップモデル（Stripeの Celtic Bank/YouLendモデル）は参考になる
