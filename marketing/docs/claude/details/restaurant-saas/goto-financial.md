## 機能比較

### データ活用・クレジットモデル
- **GoFood注文データ**: 注文頻度・金額・ピーク時間帯・メニュー構成を分析
- **GoPay取引データ**: 決済フロー・入出金パターン・取引先分布を評価
- **Moka POSデータ**: 店舗POS売上データ（統合度は不明確だが、同一エコシステム内）
- **クロスプラットフォーム統合**: GoFood + GoPay + Moka + GoBiz のデータを横断分析
- **行動スコアリング**: 取引タイミング、加盟店カテゴリ、決済場所から行動プロファイルを構築

### 引受アプローチ
- OJK（インドネシア金融庁）認可のP2Pレンディングライセンスで運営
- プラットフォーム取引データに基づくオルタナティブクレジットスコアリング
- e-KTPのみで申請可能な簡素な審査プロセス
- 10分以内の審査完了・5分以内の資金実行という超高速プロセス

### エコシステム統合
- GoFood / GoPay / Moka POS / Midtrans / GoBiz が統合エコシステムを形成
- GoPay Merchant App（2024年7月ローンチ）でMSME向けデジタル金融を一元化
- フレキシブルな資金振込機能と不正防止機能を搭載

### レポーティング
- GoPay Merchant Appから融資状況・返済スケジュールを確認可能
- GoBiz Plusダッシュボードとの統合（詳細は不明）

## 強み（SciPOSとの比較視点）

1. **インドネシア最大のデジタルエコシステム**: GoFood + GoPay + Moka + Midtransの統合が強力
2. **Moka POS保有**: クラウドPOSを自社で持つ点はSciPOSと直接競合するが、POSデータとレンディングの統合は未成熟
3. **超高速融資実行**: 審査10分・実行5分という圧倒的なスピード
4. **巨大なアンダーバンクト市場**: インドネシアの銀行口座未保有率約50%が成長余地を提供
5. **レンディング事業の急成長**: ローン帳簿68%成長、フィンテック部門が初の通年黒字化
6. **規制認可済み**: OJKライセンスで合法的に運営

## 弱み（SciPOSとの比較視点）

1. **インドネシア集中**: 事実上インドネシア市場のみ。国際展開は限定的
2. **Moka POSとレンディングの統合が不十分**: Moka POSの店内データがレンディング審査にどの程度活用されているかは不明確
3. **デリバリー偏重の信用評価**: GoFood売上中心の評価は、店内飲食の実態を反映しない
4. **融資上限が小規模**: IDR 1億（約US$5,900）は中規模レストランには不十分
5. **Tokopedia分離の影響**: Eコマースデータの喪失がクロスプラットフォーム分析を弱体化
6. **Grab-GoTo合併リスク**: 合併実現時にGoTo Financialはスピンオフされる可能性があり、エコシステムとの統合が損なわれる恐れ

## 最近の動向（直近1年）

- **2025年通期**: フィンテック部門が初の通年黒字化（Adjusted EBITDA IDR 497B、前年の△467Bから黒転）
- **2025年Q3**: 初の四半期Adjusted Pre-Tax Profit達成。レンディング売上前年比84%増のIDR 1.0兆
- **2025年通期**: 損失73%縮小（IDR 1.5兆）、売上24%増（IDR 18.3兆）
- **2025年Q1**: GoPay取引量が前年比54%急増
- **2024年7月**: GoPay Merchant App ローンチ（MSME向けフレキシブル振込・不正防止機能）
- **2026年ガイダンス**: Adjusted EBITDA IDR 3.2〜3.4兆（前年比約60%成長）を見込む
- **Grab-GoTo合併協議**: Grabによる買収交渉が進行中（GoTo企業価値US$7B超）。GoTo Financialは日本の投資家への別途売却が検討されている。2026年1月に交渉が難航報道
- **GoPay Pinjam on TikTok Shop**: インドネシア初、TikTokエコシステム内での即時キャッシュローン提供

## 調査メモ

- GoTo Financialは、Moka POS + Midtrans + GoPay + GoFood という「POS＋決済ゲートウェイ＋Eウォレット＋デリバリー」の全レイヤーを持つ数少ないプレイヤー
- しかし、これらのデータ統合はまだ完全ではなく、Moka POSの店内データがレンディング審査に直接活用されているかは確認できなかった
- SciPOSにとっての示唆：POSデータを活用したレンディングは「やろうとしている」が「完全にはできていない」プレイヤーが多い。ここに差別化の余地あり
- Grab-GoTo合併が実現すれば、GoTo Financialはスピンオフされる可能性が高く、エコシステムとの統合が弱まる可能性がある
- インドネシア市場は巨大だが、規制環境が複雑。OJKのP2Pレンディング規制は厳格化の傾向
- GoTo Financialの黒字転換は、プラットフォームデータを活用したレンディングモデルの有効性を実証
