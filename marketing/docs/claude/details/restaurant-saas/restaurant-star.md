## 強み / 弱み（SciPOSとの比較視点）

### 強み
1. **飲食チェーン特化** — チェーン店の「ファン倶楽部運営」に徹底フォーカス
2. **オリジナルアプリ制作** — 各チェーンの独自ブランドアプリを制作（汎用アプリではない）
3. **驚異的な継続率99.2%** — 導入後のチャーンが極めて低い
4. **運用サポートの充実** — 「二人三脚」でアプリ運用を支援。飲食店の多忙さを理解
5. **自動マーケティング** — 誕生日・休眠顧客への自動配信でリピーター創出
6. **第24回外食産業貢献賞受賞** — 業界での高い評価

### 弱み
1. **チェーン向け限定** — 個人店・小規模店舗には不向き
2. **料金が高い可能性** — カスタムアプリのため、小規模には割高
3. **予約管理・テーブル管理は非対応** — POSや予約システムとは別カテゴリ
4. **POS連携が限定的** — 予約・注文のコア機能ではない
5. **アプリインストールが必要** — LINEミニアプリと比較して導入ハードルが高い
6. **LINEエコシステムとの統合が弱い可能性**

### SciPOSとの差別化ポイント
- レストランスターは「CRM/ロイヤリティ」に特化。SciPOSは「POS + セルフオーダー」。直接競合ではなく補完的
- SciPOSがチェーン向けCRM/ロイヤリティ機能を強化する場合、レストランスターのアプローチ（自動化マーケティング・ファン倶楽部運営）は参考になる

## 調査メモ

- レストランスターは「アプリ開発会社」であると同時に「運用コンサルティング会社」。飲食チェーンに対して開発 + 運用を一体提供する点が強み
- 継続率99.2%は驚異的。一度導入した企業がほぼ離脱しないビジネスモデル
- 代表の磯島氏はJCB・アクセンチュア出身。決済 + コンサルティングのバックグラウンド
- ベトナムに開発拠点を持つことでコスト効率を実現
- SciPOSとは競合関係にはないが、日本のチェーン飲食店がCRM/ロイヤリティに求めるものを理解する上で有用な参考事例
