## 強み / 弱み（SciPOSとの比較視点）

### 強み
1. **順番待ちシステムのNo.1** — 約10万施設導入。順番待ち管理では圧倒的シェア
2. **巨大な会員基盤** — 累計5,770万人の会員。集客力が高い
3. **マルチ業種展開** — 飲食・医療・美容等、幅広い業種で利用可能
4. **消費者向けプラットフォーム** — EPARK自体が集客プラットフォームとして機能
5. **LINEミニアプリ連携** — 呼出機能等の新機能追加

### 弱み
1. **予約管理の深さではトレタ・ebicaに劣る** — 順番待ち特化のイメージが強い
2. **POS機能なし** — 予約/順番待ち特化（→ SciPOSはPOS一体型）
3. **グルメサイトコントローラー機能なし** — 複数グルメサイトの一元管理では弱い
4. **料金体系が不透明** — 個別見積もりのみ
5. **高級店向けではない** — カジュアル・日常利用店舗向け
6. **セルフオーダー非対応**

### SciPOSとの差別化ポイント
- EPARKの順番待ち機能はSciPOSに参考になる（ウォークイン管理の強化）
- SciPOSはPOS + セルフオーダーの一体型で明確に差別化
- EPARKのマルチ業種展開はSciPOSの飲食店特化と対照的

## 調査メモ

- EPARKは「順番待ち」に特化したプラットフォームであり、飲食店専用の予約管理（テーブル管理・顧客CRM）ではトレタやebicaの方が深い
- 5,770万人の会員基盤は巨大だが、飲食店限定ではなく医療・美容等を含む合計値
- グループ会社が多数あり（Empower Healthcare、iFLAG、EPARK Financial Partners等）、組織構造が複雑
- SciPOSとの直接競合は限定的。ターゲット・機能領域が大きく異なる
