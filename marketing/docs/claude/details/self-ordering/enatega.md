## 5. 主要プロダクト・機能一覧

### プロダクト体系

Enategaは2つの主要プロダクトラインを提供:

| プロダクト | 概要 |
|-----------|------|
| **Enatega Single Vendor** | 単一レストラン向け。自社デリバリー管理 |
| **Enatega Multi Vendor** | マルチベンダーマーケットプレイス。Uber Eats/DoorDash型 |

### アプリ構成（各プロダクト共通）

| アプリ | 概要 |
|-------|------|
| Customer App（iOS/Android） | 注文・追跡・決済 |
| Restaurant/Vendor App | 注文管理・メニュー管理 |
| Rider/Driver App | 配達管理・ルート最適化 |
| Admin Dashboard（Web） | 全体管理・分析・設定 |

## 6. 料金体系

### オープンソース vs 商用ライセンス

| モデル | 内容 |
|--------|------|
| オープンソース（GitHub） | フロントエンドコード無料公開。バックエンド/APIは非公開 |
| 商用ライセンス | バックエンドAPI + フルソースコード取得。一括購入 |

### 価格帯

| プラン | 価格 | 内容 |
|--------|------|------|
| Single Vendor | **$299〜** | 単一レストラン向けフルライセンス |
| Multi Vendor | **$4,999〜$10,000** | マルチベンダーフルライセンス |
| カスタム開発 | 要見積 | Ninjas Codeによるカスタマイズ開発 |

| 項目 | 内容 |
|------|------|
| 課金モデル | **一括ライセンス購入**（月額SaaSではない） |
| 月額費用 | なし（ホスティングは自己負担） |
| コミッション | 0% |
| サポート | 有償（カスタム開発はNinjas Codeに依頼） |
| HW費用 | なし（自社インフラ） |

**備考**: 一括購入でコード所有権を取得する「Build vs Buy」モデル。月額SaaSと異なり、長期的にはコスト効率が高いが、初期投資と技術力が必要。

## 10. 強み（SciometaPOS比較視点）

1. **オープンソース・フル所有権**: コードを完全に所有でき、ベンダーロックインなし。技術力のあるチームには魅力的
2. **極めて低い長期コスト**: 一括購入後は月額なし。SaaS型と比較して数年で元が取れる
3. **高いカスタマイズ性**: フロントエンドが完全オープンソースのため、自由にカスタマイズ可能
4. **マルチバーティカル対応**: 食品だけでなく、食料品・クーリエ・ホームサービスにも対応
5. **ライダー/ドライバー管理**: 配達員管理アプリが標準内蔵
6. **React Nativeベース**: クロスプラットフォーム開発で開発効率が高い

## 11. 弱み（SciometaPOS比較視点）

1. **POS完全非対応**: POSレジ機能なし。レストラン運営の中核ニーズを満たさない
2. **KDS・キオスク非対応**: キッチン連携やインストアセルフオーダーの機能なし
3. **シフト・勤怠・在庫管理なし**: SciometaPOSの統合運営管理と比較して大きな欠如
4. **技術力が必要**: セットアップ・運用に開発者が必要。非技術者には導入困難
5. **サポート品質不明**: 小規模チームのため、企業向けサポートの品質が未知数
6. **VC未調達**: 資金力不足で大規模な機能開発や市場拡大が制限される可能性
7. **ロイヤルティ・マーケティング機能が弱い**: CRM/ロイヤルティは基本的な顧客管理のみ
8. **デリバリー特化**: イートイン・テーブル注文よりデリバリー/テイクアウトに特化

## 13. SciometaPOSへの示唆

- **競合脅威度**: 極めて低。POS非搭載・デリバリー特化のため、SciometaPOSとは異なるセグメント
- **オープンソース戦略の参考**: フロントエンドオープンソース + バックエンドプロプライエタリのハイブリッドモデルは、コミュニティ構築の参考になる
- **新興国向け価格戦略**: 一括ライセンスモデルは、月額課金が困難な新興国市場での戦略参考
- **マルチバーティカル展開**: 食品以外のバーティカル（食料品、クーリエ等）への展開可能性の示唆
