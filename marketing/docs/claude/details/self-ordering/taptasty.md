## 5. 主要プロダクト・機能一覧

### プロダクト体系

TapTastyはレストラン向けフルスタック管理プラットフォーム。注文・オペレーション・マーケティングを一元化するオールインワンソリューション。

### 主要モジュール
1. **POS** - レストランPOSシステム
2. **モバイルアプリ** - ブランドネイティブiOS/Androidアプリ
3. **セルフオーダーキオスク** - 店内セルフ注文端末
4. **QRテーブルオーダー** - テーブルQRコードスキャン注文
5. **KDS** - キッチンディスプレイシステム
6. **在庫管理** - 統合在庫管理
7. **デリバリーアプリ** - ドライバー向けデリバリー管理アプリ
8. **ウェブサイト** - ブランドオンライン注文サイト

### 機能一覧

| 機能カテゴリ | 対応状況 | 詳細 |
|-------------|---------|------|
| **POS レジ** | ◎ | フルPOSシステム。キオスク・QR注文がPOSに自動挿入 |
| **オンライン注文** | ◎ | ブランドウェブサイト + ネイティブモバイルアプリ経由 |
| **QRコードオーダー** | ◎ | テーブルQR注文。複数人が同一QRスキャンで合算注文可。ジオロケーション制限（店内のみ） |
| **セルフオーダーキオスク** | ◎ | ブランドカスタマイズ可能。リピーター認識＆パーソナライズドプロモーション |
| **KDS** | ◎ | 左→右・上→下表示。タイマー・優先度・注文ID順。セクション分割（バー/ピザ/パスタ等）。シェフ作業時間・調理時間等4種レポート |
| **在庫管理** | ◎ | POS連携リアルタイム追跡。低在庫アラート、自動発注、レシピ管理（材料・分量）、廃棄削減 |
| **シフト・勤怠** | × | 非対応（確認限りでは明示なし） |
| **予約** | × | テーブル予約機能の明示なし |
| **CRM・ロイヤルティ** | ◎ | ロイヤルティポイント（キオスク・アプリ間で共通）。電話番号/QRコードでポイント連携。パーソナライズドプロモーション |
| **デリバリー連携** | ○ | 自社デリバリーアプリ（ドライバー向け）。外部アグリゲーター連携あり |
| **会計連携** | × | 直接的な会計ソフト連携情報なし |
| **レポート** | ○ | 注文・売上アナリティクス。KDSレポート（シェフ作業時間、調理時間、平均調理時間） |
| **多言語** | △ | KDSはルーマニア語・英語対応。その他言語対応範囲は不明 |
| **オフライン** | × | 情報なし（クラウドベースと推定） |
| **AI・自動化** | × | AI機能の明示なし |
| **マーケティング** | ○ | プッシュ通知、プロモーション、顧客エンゲージメントツール |

## 10. 強み（SciometaPOS比較視点）

| 強み | 詳細 |
|------|------|
| **フルスタック統合** | POS+キオスク+QR+KDS+在庫+モバイルアプリ+デリバリーを単一プラットフォームで提供。SciometaPOSと最も類似した構成 |
| **ネイティブモバイルアプリ** | ブランドiOS/Androidアプリ。WebViewではなくネイティブ |
| **KDSが充実** | セクション分割、優先度管理、4種のKDSレポート |
| **在庫管理が本格的** | レシピ管理・自動発注・低在庫アラート。基本的な品切れ管理以上 |
| **キオスクのパーソナライゼーション** | リピーター認識、パーソナライズドプロモーション |
| **QRオーダーのジオフェンシング** | 店内でのみ注文可能。不正注文防止 |
| **複数人同一QR注文** | 同テーブルの複数人が同じQRスキャンで合算注文 |
| **少人数で高機能** | 11名でフルスタック開発。開発効率が極めて高い |
| **東欧の開発コスト優位** | ルーマニア拠点で開発コストが低い |

## 11. 弱み（SciometaPOS比較視点）

| 弱み | 詳細 |
|------|------|
| **地域限定** | ルーマニア中心。国際展開は初期段階 |
| **シフト管理なし** | スタッフ管理・勤怠機能が欠如（SciometaPOSの強み） |
| **予約機能なし** | テーブル予約が非対応 |
| **多言語対応が限定的** | ルーマニア語+英語のみ確認。アジア言語非対応 |
| **料金非公開** | 価格透明性が低い。SMBにとって導入検討のハードルに |
| **自己資金・小規模** | VC未調達・11名。急速なスケーリングに制約 |
| **AI機能なし** | AI・自動化機能の実装なし |
| **会計連携なし** | 会計ソフトとの直接連携情報なし |
| **外部POS連携不明** | 自社POS以外のPOS連携情報なし |
| **市場認知度低い** | ルーマニア国外での認知度・レビューが少ない |

**SciometaPOS優位点**: TapTastyはフルスタック構成で最も類似した競合だが、シフト管理・多言語（日本語/アジア言語）・アジア市場特化でSciometaPOSが差別化。TapTastyのルーマニア限定展開に対し、SciometaPOSは日本・アジア市場に深く根差している。ただし、TapTastyの11名でのフルスタック開発は参考になる効率モデル。

## 総合評価

| 項目 | 評価 |
|------|------|
| SciometaPOSとの競合度 | **中〜高** |
| 脅威レベル | **中**（現時点は低、将来的に中） |
| 理由 | フルスタック構成（POS+キオスク+QR+KDS+在庫）がSciometaPOSと最も類似。ただし現状ルーマニア限定で日本/アジア市場との直接競合はない。SXSW 2025での国際展開意図は注視すべき。シフト管理・多言語・アジア特化がSciometaPOSの差別化ポイント |

**注目ポイント**: TapTastyは11名・自己資金でSciometaPOSと類似のフルスタック構成を構築しており、**プロダクト設計・開発効率の参考対象**として非常に価値が高い。特にKDSのセクション分割、キオスクのリピーター認識、QRのジオフェンシング等の機能は参考になる。
