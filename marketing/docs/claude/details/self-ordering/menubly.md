### QRオーダーの詳細フロー

1. **QRコード生成**: 管理画面で無料QRコードを生成。無制限に作成可能
2. **設置**: テーブル、カウンター、店頭にQRコードを印刷して設置
3. **スキャン/リンク共有**: 顧客がQRスキャンまたはSNSのリンクからアクセス
4. **メニュー閲覧**: モバイル最適化されたインタラクティブメニュー表示。グリッド/リスト/カテゴリビューを選択可能
5. **カスタマイズ**: サイズ、トッピング、フレーバー等のカスタムアドオンを選択
6. **注文タイプ選択**: ダインイン / ピックアップ / デリバリーを選択
7. **決済**: 100+の決済方法から選択して支払い。コミッションゼロ
8. **通知**: 店舗に注文通知。オンライン注文ダッシュボードで管理

**特徴**: メニュー更新してもQRコードの再印刷不要。30分で初期セットアップ完了。PDF/画像メニューからの変換サポートあり。

## 6. 強み / 弱み（SciPOSとの比較視点）

### 強み
- **超低コスト**: 永久無料プラン + $9.99/月のProプラン。飲食業界で最も安価な部類
- **ゼロコミッション**: 注文手数料ゼロは小規模店舗に非常に魅力的
- **簡単セットアップ**: 30分で完了。技術知識不要
- **ウェブサイトビルダー統合**: メニュー + ウェブサイト + QRコード + オンライン注文を一元提供
- **SEO最適化**: 組み込みSEOツールでレストランの検索可視性を向上
- **Link-in-bio最適化**: Instagram等のソーシャルメディアからの集客に最適化
- **100+決済方法**: Stripe経由で幅広い決済に対応
- **WhatsApp注文**: WhatsApp統合は東南アジア・中東市場で有効
- **PDF変換サービス**: 既存メニューからの移行が容易
- **豊富なブログコンテンツ**: SEO戦略として競合比較記事を大量に公開。コンテンツマーケティングに注力

### 弱み（SciPOSの優位点）
- **POS機能がない**: POSシステムではなく、メニュー表示・注文ツール。SciPOSは統合POS
- **キオスク非対応**: セルフオーダーキオスク機能なし
- **KDS未提供**: キッチンディスプレイシステムなし
- **テーブル管理なし**: テーブルレイアウト・予約管理機能なし
- **在庫管理なし**: 在庫・仕入れ管理機能が存在しない
- **CRM/ロイヤルティなし**: 顧客管理・ロイヤルティプログラムなし
- **シフト管理なし**: スタッフ管理・シフト機能なし
- **分析機能が限定的**: 基本的な分析のみ。詳細なビジネスインテリジェンスは提供しない
- **POS連携が不明確**: 対応POSシステムの具体的なリストが公開されていない
- **スケーラビリティに疑問**: シングルページウェブサイトの設計は小規模店舗向け
- **会社情報の不透明さ**: 創業年、本社、チーム規模等が非公開。信頼性に影響
- **レビューが極少**: Capterra上で1件のレビューのみ。実績の検証が困難
- **多言語対応が不明**: 具体的な対応言語数が非公開

## 8. 調査メモ

- Menublyは「レストランのLinktree」のような位置づけ。シングルページウェブサイト + デジタルメニュー + QRコード + オンライン注文をセットで提供
- SciPOSとの直接競合度は非常に低い。Menublyはメニュー表示・簡易注文ツールであり、レストラン管理SaaSではない
- 最大の強みは「無料」と「簡単さ」。技術リソースのない個人飲食店のエントリーポイントとして機能
- 競合比較ブログ記事を大量に公開するSEO戦略は、コンテンツマーケティングの参考になる
- 会社情報がほぼ非公開であることは懸念材料。長期的なサービス継続性のリスクあり
- WhatsApp注文統合は中東・東南アジア市場では有効。SciPOSへの機能追加を検討する価値あり
- $9.99/月という価格帯は、SciPOSの価値提案を「フル統合プラットフォームの価値」として明確に差別化する材料になる
