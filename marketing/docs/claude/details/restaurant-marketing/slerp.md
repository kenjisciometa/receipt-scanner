## 機能比較

| 機能カテゴリ | 対応状況 |
|---|---|
| **POSレジ機能** | 間接対応。独自POS非保有。Deliverect・3S POS等との連携で既存POSへ注文自動転送 |
| **セルフオーダー（キオスク）** | 非対応 |
| **セルフオーダー（テーブル／QR）** | 対応。QRコードによるテーブルオーダー（スキャンして注文・決済完結） |
| **KDS（キッチンディスプレイ）** | 間接対応（POS連携経由） |
| **在庫管理** | 非対応 |
| **シフト・勤怠管理** | 非対応 |
| **予約管理** | 限定対応。事前注文・ケータリング注文の予約受付は可能 |
| **CRM / ロイヤルティ** | 対応（強み）。スタンプカード・ポイント・ティアシステム、Push通知、顧客セグメンテーション、バースデーキャンペーン、休眠顧客自動リエンゲージメント |
| **デリバリー連携** | 対応（強み）。Slerp Dispatch：Uber Direct・Deliveroo Express・Stuart・Addison Lee・Onfleet等と統合 |
| **会計ソフト連携** | 非明示 |
| **レポート・分析** | 対応。AOV・リピート率・顧客行動データのダッシュボード |
| **多言語対応** | 非明示（英語のみ） |
| **オフライン対応** | 非対応（クラウド型） |
| **AI / 自動化機能** | 対応（拡張中）。AIサポートCRM、自動メール・SMSトリガー送信、AIマーケティングキャンペーン提案 |

### 主要機能詳細
- **ブランドアプリ（iOS/Android）**: 飲食ブランドのロゴ・カラーで完全カスタマイズされたモバイルアプリ
- **Order with Google連携（2024年5月）**: Googleビジネスプロフィールから直接注文可能
- **ウェブ注文**: ブランデッドウェブストア（独自ドメイン対応）
- **ケータリング注文管理**: 法人・大口向け事前注文管理
- **クリック＆コレクト**: 店頭受取予約機能

---

## ハードウェアモデル
- **専用端末 vs BYOD vs 両対応**: 完全BYOD型。独自ハードウェア非提供
- **対応決済端末**: Stripe等の決済ゲートウェイをオンライン注文に統合
- **プリンタ対応**: POS連携経由でキッチンへの印刷はPOS側で処理

---

## 強み / 弱み（SciPOS との比較視点）

### 強み
1. **0%コミッション・固定月額モデル**: マーケットプレイスへの依存断ち切り。£100k以上の年間コスト削減事例
2. **高い転換率**: 業界平均の7.1倍、マーケットプレイス比32%高い注文単価
3. **デリバリー統合（Slerp Dispatch）**: 複数クーリエを一元管理・自動アサイン
4. **CRM・ロイヤルティ・アプリ一体型**: 注文管理・顧客データ・ロイヤルティ・メールマーケティング・Push通知を単一プラットフォーム
5. **AI強化CRM（2023年〜）**: 自動トリガーメール・SMS
6. **Capterra 2025年「Best Ease of Use」受賞**
7. **Order with Google連携**
8. **プレミアム飲食ブランドとの実績**: Ottolenghi・Eataly等

### 弱み
1. **POSレジ機能なし**: 既存POSシステムとの連携が必須
2. **キオスク非対応**
3. **在庫・シフト管理なし**
4. **オフライン動作不可**
5. **イギリス市場特化**: グローバル展開実績なし
6. **ハードウェアなし**
7. **料金非公開**
8. **小規模チーム・資金**: 従業員約40名、売上$4.4M
