## 料金体系

課金単位は**1ユーザーあたり/月**。チーム規模に応じてスケールする料金モデル。

| プラン | 月額 | 主な機能 |
|--------|------|---------|
| Essentials | $2.50/ユーザー/月 | シフトスケジュール作成・共有、タイムクロック（打刻）、チームメッセージング |
| Pro | $5.00/ユーザー/月 | Essentials + カスタムシフトテンプレート、詳細スケジュールルール設定 |
| Premium | $8.00/ユーザー/月 | Pro + API アクセス、SSO/SAML認証、カスタムレポート |

### 追加オプション

| オプション | 月額 |
|-----------|------|
| 勤怠管理（Time & Attendance）アドオン — Essentials | +$1.50/ユーザー/月 |
| 勤怠管理（Time & Attendance）アドオン — Pro/Premium | +$2.00/ユーザー/月 |

- **年間払い割引**: あり（詳細非公開）
- **無料トライアル**: 14日間
- **無料プラン**: なし
- **ハードウェア費用**: 非該当（SaaS専業。スマートフォン・タブレットはBYOD）

### 料金の特徴
- 業界最安水準の価格設定（Essentials $2.50/ユーザー/月〜）
- ユーザー数無制限（全プラン）
- 勤怠管理が別課金（基本プランにはスケジュールのみ）

## ターゲット市場

### 対応業態
レストラン（QSR・フルサービス）、小売、ホスピタリティ（ホテル）、ヘルスケア、イベント、物流・倉庫、教育機関。時間制（hourly）ワーカーを擁する全業態が対象。

### 対象規模
- **SMB（5〜500名）**: 主力ターゲット。個人店舗から中規模チェーンまで
- **エンタープライズ**: Premium プランでSSO/API対応。ただし大規模エンタープライズ向け機能は限定的

### 主要地域
米国（最大市場）、カナダ。グローバル50か国以上で利用されているが、コンプライアンス対応（労働法等）は米国中心。

## 強み / 弱み（SciPOS との比較視点）

### 強み
1. **圧倒的な低価格**: $2.50/ユーザー/月〜は業界最安水準。小規模飲食店が導入しやすい価格帯
2. **モバイルファーストUX**: 直感的なモバイルアプリ。従業員のセルフサービス（シフト確認・交換・休暇申請）が充実
3. **導入の手軽さ**: SaaS専業でハードウェア不要。14日間トライアルで即開始可能
4. **豊富なペイロール連携**: ADP、QuickBooks、Gusto、Rippling、Paychex等の主要米国給与サービスと直接連携
5. **大規模ユーザーベース**: 150,000+事業所、100万人以上のユーザーによる実績とブランド認知
6. **管理者の時間節約**: 週15時間のスケジュール管理時間削減（公称値）
7. **Auto Scheduler**: 従業員の適格性・可用性を考慮した自動スケジュール生成

### 弱み
1. **シフト管理特化（POS・注文管理なし）**: レストラン運営に必要なPOS、セルフオーダー、KDS、在庫管理等は一切なし。SciPOSの統合プラットフォームと比較して、別途POS導入が必須
2. **勤怠管理が別課金**: 基本プランにタイムクロックが含まれず、追加費用が発生。実質的なコストは表示価格より高い
3. **米国中心の設計**: 労働法コンプライアンス、給与連携ともに米国市場向け。欧州・日本の労働法規（36協定等）への対応は不十分
4. **多言語対応の不足**: 英語中心。日本語対応なし。多国籍スタッフを抱える飲食店には不向き
5. **アプリの安定性問題**: 複数ユーザーの同時打刻時にフリーズする報告あり。ピークタイムのシフト交代時に問題
6. **コミュニケーション機能の限界**: 1対1メッセージとグループアナウンスのみ。スレッド型会話やメッセージピン留め等の高度な機能なし
7. **カスタマーサポート**: 電話サポートなし。オンラインチケット・ライブチャットのみで対応時間も限定的
8. **リアルタイム労働時間表示なし**: アクティブシフト中の累計労働時間が表示されず、残業管理が困難
9. **カスタマイズ性の限界**: UIは直感的だがカスタマイズ性に乏しい。複雑な組織構造やシフトルールへの対応が難しい

### SciPOS との差別化ポイント
- **SciPOS**: POS + セルフオーダー + KDS + 在庫 + シフト管理を統合。飲食店が1つのプラットフォームで運営可能
- **When I Work**: シフト管理・勤怠のみ。飲食店は別途POS（Square、Toast等）の導入が必要で、システム間のデータ連携コスト・運用コストが発生
- **多言語・多地域**: SciPOSは日本語・多言語対応。When I Workは英語中心で日本市場への適合性が低い
- **業態特化 vs 汎用**: When I Workは業態を問わない汎用ツール。SciPOSは飲食業態に最適化された機能セット

## 総合評価

When I Workは低価格・モバイルファーストの汎用シフト管理ツールとして、特に米国のSMB市場で強いポジションを持つ。しかし、飲食業態に特化した機能（POS、セルフオーダー、KDS、在庫管理）は一切持たず、シフト管理のみの単機能ツールである。SciPOSの統合型プラットフォーム戦略とは根本的に異なるアプローチであり、直接的な競合というよりは、SciPOSのシフト管理機能が代替し得る「置き換え対象」として位置づけられる。日本・欧州市場においては多言語対応・労働法コンプライアンスの面でSciPOSが明確な優位性を持つ。
