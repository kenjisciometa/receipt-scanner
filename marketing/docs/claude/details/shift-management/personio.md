## 料金体系

課金単位は**1従業員あたり/月（PEPM: Per Employee Per Month）**。3プラン構成 + アドオンモジュール。

| プラン | 月額 | 主な内容 |
|--------|------|---------|
| Core | $2.96〜/従業員/月（公開最低価格） | 従業員データベース、文書管理、休暇・欠勤管理、基本ワークフロー |
| Core Pro | カスタム見積 | Core + 無制限法人管理、ポジション管理、拡張文書管理、人員計画、OAuth / Okta / Microsoft Entra ID 連携 |
| Professional | カスタム見積 | Core Pro + 採用管理、業績評価、高度なレポート・分析、従業員セルフサービス |
| Enterprise | カスタム見積 | Professional + 高度な自動化、カスタムワークフロー、APIアクセス、専任サポート、セキュリティ強化、マルチカントリー対応 |

- **初期費用**: 非公開（導入支援・オンボーディング込みの見積制）
- **無料プラン**: なし
- **無料トライアル**: あり（デモリクエスト形式）
- **ボリュームディスカウント**: 100名以上で割引適用が一般的
- **ハードウェア費用**: 非該当（SaaS専業。BYOD前提）
- **対応企業規模**: 従業員10〜5,000名
- **注意点**: 料金は非公開で個別見積制。従業員数・選択モジュール・契約条件により変動

## ターゲット市場

### 対応業態
全業種対応の汎用HRプラットフォーム。飲食・ホスピタリティに特化していないが、ホテル・ケータリング業向けに季節的な大量採用、シフト管理、給与処理への対応を訴求。主な対象業種: オフィス業務、テクノロジー企業、小売、製造、サービス業、ホスピタリティ全般。

### 対象規模
- **SMB（中小企業）**: 主力市場。従業員10〜200名規模が最多
- **ミッドマーケット**: 200〜2,000名規模に拡大推進中
- **エンタープライズ**: 最大5,000名規模まで対応。Oracle / SAP / Workday との競合領域では限定的

### 主要地域
欧州が中心。ドイツ（本国、特に給与計算で強み）、英国、アイルランド、オランダ、スペイン、オーストリア。米国（ニューヨーク）にも拠点があるが、給与計算は欧州市場が主力。70か国以上にユーザーが存在するが、ローカライズが深い地域は欧州中心。

### HR関連コア機能（シフト管理以外）
- **採用管理（ATS）**: 求人掲載、応募者追跡、採用パイプライン管理。2026年にaurio買収でAI候補者スクリーニングを強化
- **給与管理**: 欧州市場（特にドイツ・英国・オーストリア・スペイン）に強い。YoY 80%成長、1,500社以上が利用。米国給与はネイティブ非対応
- **オンボーディング / オフボーディング**: 構造化されたワークフロー、タスク管理、文書署名
- **業績評価**: 目標設定、評価サイクル、1on1、AIによるフィードバック要約
- **文書管理**: HR文書の一元管理、電子署名（e-signature）
- **従業員サーベイ**: エンゲージメント調査、パルスサーベイ
- **従業員セルフサービス**: 休暇申請、文書閲覧、個人情報更新

## 強み / 弱み（SciometaPOS との比較視点）

### 強み
1. **欧州最大級のHR SaaSプラットフォーム**: 評価額 $8.5B、顧客15,000社以上、1,757名の従業員。欧州HR市場における圧倒的なブランド力と資金力。
2. **People Operating Systemとしての統合性**: 採用・オンボーディング・勤怠・給与・業績評価・文書管理をワンプラットフォームで提供。HR部門にとっての統合価値が極めて高い。
3. **欧州給与計算の深い対応**: 特にドイツ（DATEV連携）を筆頭に、英国・オーストリア・スペインの給与計算にネイティブ対応。給与事業はYoY 80%成長と急拡大中。
4. **AI戦略の積極推進**: AI Assistant（全従業員向け）、Smart Automations、AIレポート生成に加え、2026年4月にaurio買収でAI採用機能を統合。黒字化と同時にAI投資を加速。
5. **黒字化達成（2026年4月）**: 創業11年で黒字化を発表。持続可能な成長モデルとして競合に対する信頼性が向上。
6. **豊富なインテグレーション**: Marketplace経由で多数のサードパーティ連携（Planday、Quinyx、shyftplan等のWFMツール含む）を提供。

### 弱み
1. **飲食業への特化度が極めて低い**: 汎用HRプラットフォームであり、飲食業固有のニーズ（POS連携、KDS、セルフオーダー、在庫管理、テーブル管理）は一切対応なし。レストラン特化型の7shifts、HotSchedules、Skelloと比較すると業界フィット度で大きく劣る。
2. **シフト管理機能が基本レベル**: シフト管理は提供しているが、複雑なスケジュール（分割シフト、変形労働時間、飲食業の繁忙期対応等）への柔軟性が不足。Planday・shyftplan等との連携で補完する設計。
3. **カスタマイズ性の制約**: 設定オプションが限定的で、複雑なHRプロセスや業界固有の要件に対応しにくい。
4. **モバイルアプリの機能制限**: デスクトップ版と比較してモバイルアプリの機能が制限されている。飲食業のように現場スタッフがPCを使わない環境では不利。
5. **日本・アジア市場への対応なし**: 日本語非対応、日本の労働基準法（36協定、変形労働時間制等）への対応なし。アジア市場への展開実績もない。
6. **料金の不透明性**: 完全カスタム見積制で事前の価格比較が困難。小規模飲食店にとってはコスト予測が難しい。
7. **レポート・分析の制限**: ユーザーレビューでレポートのカスタマイズ性やダッシュボードの柔軟性に不満が見られる。

### SciometaPOSとの競合関係
- **直接競合: 極めて低い** --- Personioは汎用HRプラットフォーム、SciometaPOSはPOS+セルフオーダー+シフト管理の飲食業特化型統合システム。ポジショニングが根本的に異なる。
- **間接競合**: SciometaPOSのシフト管理モジュールとは機能面で一部重複する。特に欧州市場でPersonioを既に導入している飲食店が勤怠管理を追加購入するケースでは、SciometaPOSのシフト管理単体での訴求はやや弱い。
- **差別化ポイント**: SciometaPOSの最大の強みは「POS + セルフオーダー + KDS + 在庫 + シフト管理」が一つのプラットフォームに統合されている点。飲食店オーナーにとって、Personio + 別途POS + 別途セルフオーダー + 別途在庫管理という複数SaaS契約よりも、SciometaPOSのワンストップ体験が圧倒的に優位。
- **連携の可能性**: 大規模チェーンが既にPersonioをHR基盤として導入している場合、SciometaPOSのPOS/セルフオーダー/シフト機能との連携（API経由の勤怠データ・従業員データ連携等）が戦略的選択肢になりうる。PersonioのMarketplace連携モデルを活用したパートナーシップも検討余地あり。

## 最近の動向（直近1年）

1. **黒字化達成 + aurio買収（2026年4月）**: 創業11年で初の黒字化を発表。同時にミュンヘン拠点の採用AIスタートアップ aurio を買収し、AI採用機能（候補者ソーシング・スクリーニングの自動化）を統合。2026年後半に顧客向けローンチ予定。
2. **給与事業の急成長**: 給与管理事業がYoY 80%成長を達成し、1,500社以上が利用。欧州給与計算市場でのプレゼンスを拡大中。
3. **AI Assistant の全従業員展開（2025年Q4）**: Personio AI Assistantを全従業員に開放。自然言語でのHR質問応答、リアルタイムデータ照会が可能に。HR部門の管理負荷軽減を推進。
4. **Smart Automations のリリース（2025年Q3-Q4）**: 繰り返しタスクの自動検出・ワークフロー提案・リマインダー機能を搭載。受信トレイの優先順位付けなど、HR業務の自動化を強化。
5. **HUG 2025 での大型プロダクトアップデート（2025年）**: 創業10周年を記念し、Core HR・Payroll・Performance & Development・Surveys・Analytics・Reportingの各領域で大幅機能強化を発表。新ブランドアイデンティティも同時公開。
6. **$25.9M 追加調達（2025年6月）**: Series E の追加ラウンドとして $25.9M を調達。AI開発と製品拡張に充当。
